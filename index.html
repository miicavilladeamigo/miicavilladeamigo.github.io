Introducción

Estos son los trabajos que se hicieron durante el año en la comisión 4 con el profesor Leonardo Garay.
Los objetivos de estos trabajos, era demostrar los conocimientos enseñados en clase durante todo el ciclo
electivo del presente año. Entre los conocimientos enseñados, se encuentran los condicionales, navegacion
(con teclado y mouse), ciclos FOR, arreglos, funciones (encapsulamiento de codigo y reutilizables), programación
orientada a objetos, etc. A lo largo del año, fuimos haciendo trabajos semanales, para comprender como
funcionaba el concepto de programación orientada a objetos, ya que el TP5 debiamos crear un videojuego
usando dicho tema. Todos estos temas, son utilizados en el trabajo final, donde se debía incorporar la aventura
gráfica (tp3 y t4) con el tp5, porque el videojuego debía estar relacionado con la avenutura (se debía reemplazar
una pantalla de la historia). El tp2, solo demostraba como se lograba agregar movilidad a los conceptos,
que posteriormente fueron abordados en el tp3, tp4 y tp final.

Trabajo práctico Nº3 aventura gráfica

Para este trabajo, debiamos elegir una historia lineal y convirtirla en una aventrua gráfica. Las aventuras gráficas, se las
conoce por tener que tomar desiciones a lo largo de la historia. Dependiendo de las decisiones que se eligan, el final puede
variar. En este caso, elegimos la fabula de los tres hermanos, escrita por J.K Rowling para el sexto libro de la saga de Harry Potter.
Esta cuenta sobre tres hermanos que se encontraban viajando, hasta que se cruzan con la muerte. Esta, les otorga unos regalos por
ser asustos para evadirla. El hermano mayor recibe la varita mágica más poderosa, el hermano mediano recibe una piedra
para traer los muertos devuelta a la vida y el hermano menor, recibe un manto de invisibilidad. El hermano mayor es asesinado
por otro mago (esto es producto de su avaricia).
El hermano mediano se suicida por nostalgia, ya que intento traer de nuevo a la vida a su novia, pero al no pertenecer en la
tierra de los vivos, se encuentra distante por el velo.
El hermano menor, se oculto de la muerte por años. Pero cuando llega a una edad avanzada, se quita la capa, recibiendo
a la muerte como una vieja amiga.
En cuanto a los conceptos de programación, se encuentran las funciones que sirven para encapsuar código, diferentes tipos
de variables (del tipo entero,para fuetes tipograficas, imagenes) y condicionales (para la navegacion y los diferentes estados
de pantalla).

Trabajo práctico Nº5 videoJuego

En base al trabajo anterior, se debía realizar de forma independiente, un minijuego, que debía
estar relacionado con la aventura. Este trabajo fue realizado con la programación Orientada a Objetos.
Este juego, se trata de derrotar al mago con el que se enfrenta el hermano mayor. Utilizamos las clases:

Jugador (donde se cargaron las imagenes de este y sus hechizos de defensa)
Encantamientos (los encantamientos enemigos)
Hechizo (carga los hechizos que dispara el Jugador)
Vidas (dibuja el contador de vidas)
Puntos (dibuja la cantidad de puntos que va ganando el Jugador)
VideoJuego (se encarga de dibujar todo el videoJuego)
Juego (controla los estados del minijuego).
Esto se controla de la siguiente manera
if (status == 0) {
dibujoPantallaInicio(); --> Explica lo que se debe hacer en el videoJuego
} else if (status == 1) {
dibujarVideoJuego(); --> dibuja el juego
} else if (status == 2) {
dibujarMensajes("Has Derrotado al mago. Puedes avanzar en la historia"); --> si pierde, dibuja este mensaje
} else if (status == 3) {
dibujarMensajes("El mago te ha derrotado. Presiona una tecla para volver a intentarlo"); --> si gana, se dibuja este mensaje
}

Trabajo práctico final

En el trabajo practico final, se debía incorporar el videojuego a la aventura gráfica. A su vez, se
debía pasar el tp4 a objetos, para poder reemplazar la pantalla que corresponde al minijuego. Se
utilizaron las mimsas clases del trabajo anterior y se agregaron nuevas. Las clases nuevas son:

aventuraGrafica, que se encarga de dibujar todas las pantallas (con los parametros correspondientes),
la navegación de la misma e incluir el videojuego en la pantalla siete. A su vez, carga todas las imagenes
en un ciclo FOR para ser llamadas cuando se dibuja la pantalla.
Pantalla, recibe parametros del tipo "string" (textos, para la historia y la navegación), image (las imagenes
tienen numeros que corresponde a cada una de las pantallas. Si la pantalla es uno, entonces se va a cargar la
imagen uno), fuentes (las fuentes tipograficas varia en cada pantalla), color (una de las imagenes, contiene
un fondo claro, por lo que el texto no se lee con claridad. Optamos por cambiar el color de las letras), la
clase creditos y juego (estas solo son llamadas cuando es necesario. Si no se usan, se coloca el null).
Textos. Esta carga todos los textos de la historia, como de la navegación. Esto se encuentra cargado
en arreglos para ser llamados posteriormente. Para ser cargados en la pantalla, se coloco una variable
del tipo int que devuelve un valor (en este caso, sería el número de la pantalla). Ej:
String acomodarTexto(int pantalla) {
return textos[pantalla];
}
Lo mismo sucede con la navegación. Ej:
String acomodarNavegacion (int pantalla) {
return navegacion [pantalla];
}
Créditos. Esta clase, solo dibuja los creditos de la historia y agrega el movimiento clasico de las palabras
subiendo desde abajo hacia arriba (se incorporó la idea de que se detenga cuando 650 sea mayor que el
ancho de la pantalla dividido seis). Ej:
if (resize.resizePosY(pantalla14_PosY_Inicial) > width/6) { pantalla14_PosY_Inicial--; } }
Resize. Esta clase solo se encarga de cargar la redimensión de la pantalla. Esta es pasada como parametro
al resto de las clases cuando sea necesario. Ej
int resizePosX(int valor) {
return round(map(valor, 0, 800, 0, width));
} int resizePosY(int valor) {
return round(map(valor, 0, 600, 0, height));
}
Dentro del valor, estarian las posiciones en X e Y de los textos, navegación, etc.
